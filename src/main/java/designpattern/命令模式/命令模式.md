#命令模式（Command Pattern）
将一个请求封装成一个对象，从而可以使用不同的请求把客户端参数化，对请求排队或记录请求日志，可以提供命令的撤消和恢复功能

----------
----------
##好处：
1.类间解耦
2.可扩展性
3.和其他模式结合更有价值（责任链模式，模板方法模式）
----------
##缺点：
1.Command的子类会很多，使得类膨胀

----------  
##应用场景：
1.可以看作是命令的地方就可以使用命令模式
