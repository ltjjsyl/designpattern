#状态模式（xxx Pattern）
当一个对象内在状态改变时允许改变行为，这个对象看起来像改变了其类

----------
----------
##好处：
1.结构清晰，没有过多的if...else和switch...case
2.遵循设计原则，单一职责原则，开闭原则
3.封装性好

----------
##缺点：
1.子类太多，类膨胀

----------  
##应用场景：
1.行为随状态改变而改变的场景，eg：权限设计
2.条件/分支判断语句的代替者


----------
##注意事项
1.对象的状态最好不要超过5个
